# Техническое задание (ТЗ)
## Интернет-магазин верхней одежды

---

## 1. Введение

### 1.1. Наименование проекта
Интернет-магазин верхней одежды

### 1.2. Цель создания
Разработать веб-приложение для публичного размещения каталога товаров (верхняя одежда) с возможностью управления товарами и категориями через административную панель.

### 1.3. Целевая аудитория
- **Покупатели** — пользователи, просматривающие каталог и детальную информацию о товарах.
- **Администраторы** — сотрудники компании, управляющие ассортиментом (добавление, редактирование, удаление товаров и категорий).

---

## 2. Функциональные требования

### 2.1. Пользовательская часть (публичная)

#### 2.1.1. Главная страница
- Отображается список всех активных товаров.
- Товары выводятся с изображением, названием, ценой, размером, цветом.
- Сортировка товаров:
  - По цене (возрастание/убывание)
  - По новизне (сначала новые)
- Пагинация (по 9 или 12 товаров на странице).
- Возможность фильтрации по категориям.

#### 2.1.2. Страница категории
- Отображается название категории.
- Список товаров, относящихся к этой категории.
- ЧПУ-ссылка: `/category/muzhskie-kurtki`

#### 2.1.3. Страница товара
- Отображается полная информация о товаре:
  - Фотография
  - Название
  - Цена
  - Размеры (S, M, L, XL)
  - Цвет
  - Материал
  - Описание и характеристики
  - Количество на складе (если 0 — пометка «Нет в наличии»)
- ЧПУ-ссылка: `/product/stilnaya-kozhanka-chernyy-m`

#### 2.1.4. Поиск
- Поисковая строка в шапке сайта.
- Поиск осуществляется по полям: название товара, описание, категория.

#### 2.1.5. Навигация
- Меню категорий в шапке (Dropdown/Верхнее меню).
- Хлебные крошки (Breadcrumbs): `Главная → Категория → Товар`
- Footer с контактами (условно).

### 2.2. Административная часть

**Доступ:** По ссылке `/admin` (через таблицу пользователей Yii2).

#### 2.2.1. Управление категориями (`/admin/category`)

| Действие | Описание |
|----------|----------|
| Создание категории | Поля: название, slug (генерируется из названия), описание, изображение |
| Редактирование | Изменение всех полей, включая изображение |
| Просмотр списка | Таблица со всеми категориями (ID, название, дата создания, количество товаров, действия) |
| Удаление категории | Только если в категории нет товаров. Иначе ошибка |

#### 2.2.2. Управление товарами (`/admin/product`)

| Действие | Описание |
|----------|----------|
| Создание товара | Поля: название, категория, цена, описание, размеры, цвет, материал, количество на складе, загрузка изображения |
| Редактирование | Изменение всех полей, возможность заменить изображение |
| Просмотр списка | Таблица с фильтрами по категории, поиском по названию, пагинацией |
| Удаление товара | Удаление записи из БД и файла изображения с сервера |

#### 2.2.3. Slug (ЧПУ)
- Автоматическая генерация slug из названия на латинице.
- Уникальность slug проверяется. При повторе добавляется суффикс `-2`, `-3` и т.д.

## 3. Нефункциональные требования

### 3.1. Производительность
- Страница каталога должна грузиться не более 2 секунд при 100+ товарах.

### 3.2. Безопасность
- Защита от SQL-инъекций через AR-запросы Yii2.
- XSS-защита: экранирование вывода.
- CSRF-защита для форм.
- Доступ к админ-панели — только авторизованным пользователям.

### 3.3. Кроссплатформенность
- Адаптивная верстка (Bootstrap 5) — корректное отображение на ПК, планшетах и телефонах.

### 3.4. Требования к коду
- MVC-архитектура Yii2.
- Наличие миграций для всех таблиц.
- Использование Gii для генерации базовых CRUD.

## 4. Структура базы данных (миграции)

### Таблица `categories`

| Поле | Тип | Описание |
|------|-----|----------|
| id | PK, autoincrement | Первичный ключ |
| name | string, not null | Название категории |
| slug | string, unique, not null | Уникальный идентификатор (ЧПУ) |
| description | text | Описание |
| image | string, null | Изображение |
| created_at | timestamp | Дата создания |
| updated_at | timestamp | Дата обновления |

### Таблица `products`

| Поле | Тип | Описание |
|------|-----|----------|
| id | PK, autoincrement | Первичный ключ |
| category_id | FK → categories.id | Внешний ключ |
| name | string, not null | Название товара |
| slug | string, unique, not null | Уникальный идентификатор (ЧПУ) |
| price | decimal(10,2), not null | Цена |
| description | text | Описание |
| image | string, null | Изображение |
| size | string, null | Размеры (S,M,L,XL) |
| color | string, null | Цвет |
| material | string, null | Материал |
| stock | integer, default 0 | Количество на складе |
| created_at | timestamp | Дата создания |
| updated_at | timestamp | Дата обновления |

## 5. Сценарии использования (User Stories)

### Сценарий 1: Пользователь просматривает товары
1. Пользователь заходит на главную страницу.
2. Видит список товаров (по 9 штук).
3. Выбирает категорию «Куртки» в меню.
4. Система перенаправляет на `/category/kurtki` и показывает только куртки.

### Сценарий 2: Администратор добавляет новый товар
1. Администратор входит в админ-панель (`/admin`).
2. Переходит в раздел «Товары» → «Создать товар».
3. Заполняет поля: название «Кожаная куртка», цена 7990, категория «Куртки».
4. Загружает изображение товара.
5. Нажимает «Сохранить».
6. Система генерирует slug `kozhanya-kurtka`, сохраняет товар и выводит сообщение «Товар создан».

### Сценарий 3: Пользователь ищет товар
1. Пользователь вводит в поиск «зимняя куртка».
2. Система показывает все товары, где в названии или описании встречается фраза.
3. Если ничего не найдено — выводится сообщение «Товары не найдены».

## 6. Интерфейс и навигация

### 6.1. Публичный интерфейс
- **Шапка сайта:** логотип, меню категорий (dropdown), поисковая строка.
- **Основной контент:** список товаров в виде карточек.
- **Карточка товара:** изображение, название, цена, кнопка «Подробнее».
- **Подвал (footer):** контакты.

### 6.2. Административный интерфейс
- Используется стандартный шаблон Yii2 с меню:
  - Категории (список, создать)
  - Товары (список, создать)
- Генерация через Gii с минимальной доработкой.
